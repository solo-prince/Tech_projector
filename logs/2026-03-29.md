## 🚀 Project Update - Automated Hydroponics System (HCS-v2)

**Date:** 2026-03-29

**Progress:**
The core data acquisition pipeline for nutrient monitoring is now stable. The focus this past sprint was on integrating the new pH and Electrical Conductivity (EC) sensors, ensuring reliable data logging to the cloud platform, and implementing basic alert logic for parameter deviations.

**Technical Work:**
- **Sensor Integration:** Replaced the analog pH sensor with a new BNC-type probe (Atlas Scientific EZO-pH module) for improved accuracy and stability. Implemented a robust I2C communication protocol for both the pH and EC modules on the ESP32 microcontroller.
- **Data Pipeline:** Refactored the data acquisition loop to implement a moving average filter over 10 samples (at 1-second intervals) to mitigate noise spikes, particularly noticeable in the EC readings when pumps activate.
- **Cloud Communication:** Configured the ESP32 to send sensor data via MQTT to a local Node-RED instance, which then forwards the data to InfluxDB for time-series storage and Grafana for visualization.

**Issues & Fixes:**
- **Issue:** Experienced significant data drift and noise spikes in the pH sensor readings, especially when the main water pump (Solenoid A) was active. This was initially attributed to electrical noise from the motor.
- **Fix:** Implemented software-based noise reduction using a moving average filter. Additionally, physically moved the sensor modules away from high-current switching components and added proper grounding to minimize electromagnetic interference (EMI). The moving average filter proved highly effective, reducing variance by approximately 60%.

**Next Steps:**
- **Actuator Integration:** Begin work on the control logic for the automated dosing pumps (Peristaltic Pump A for pH Up, Peristaltic Pump B for pH Down).
- **Control Loop Implementation:** Implement a PID controller in the ESP32 firmware to manage pH and EC levels within a target range, ensuring nutrient stability for the plants.
- **UI Development:** Create a dashboard in Node-RED for manual override and setting target parameters.

## 🚀 Project Update - Delta Manipulator Arm (DMP-v1)

**Date:** 2026-03-29

**Progress:**
Successfully completed initial testing of the inverse kinematics (IK) solver. The arm can now translate a Cartesian coordinate (x, y, z) into the required joint angles for the three primary actuators. This marks a significant milestone in achieving precise movement control.

**Technical Work:**
- **Firmware Development:** Developed the IK solver using C++ on the STM32F4 microcontroller. The solution calculates the required angles based on the physical dimensions of the Delta manipulator linkages.
- **Motor Control:** Implemented a new velocity control loop (PID) for the stepper motors with integrated encoders. The encoders are essential for closed-loop feedback, ensuring the actual position matches the calculated target angle from the IK solver.
- **Communication Protocol:** Created a serial interface (UART) for receiving commands from a host PC (Python script sending target coordinates) and a feedback loop providing current joint angles and status.

**Issues & Fixes:**
- **Issue:** During initial movement tests, the arm exhibited significant mechanical backlash and vibration when changing direction quickly. The PID controller was causing overshoot and instability.
- **Fix:** Re-tuned the PID parameters. Increased the D-term (derivative gain) to dampen oscillations and reduce overshoot. Also adjusted the acceleration profile in the motion planning library to ensure smoother starts and stops, minimizing mechanical stress and improving accuracy at the target position.

**Next Steps:**
- **Path Planning:** Implement trajectory generation (spline interpolation) to move between waypoints in a smooth, continuous path rather than abrupt point-to-point movements.
- **End-Effector Integration:** Install and program the gripper mechanism (solenoid-actuated) and integrate its control into the main state machine.
- **Calibration Routine:** Develop an automated homing procedure using limit switches to define the zero-reference point for all axes on startup.

## 🚀 Project Update - AI Defect Detection System (AQC-v3)

**Date:** 2026-03-29

**Progress:**
Completed the first round of training for the new YOLOv8 object detection model. The model is successfully identifying the primary defect classes (e.g., surface scratches, missing components) on the conveyor belt. Focus shifted from model training to improving the robustness of the data pipeline and integrating the model into the production environment.

**Technical Work:**
- **Model Training:** Trained the YOLOv8-L model for 100 epochs on a dataset of 5,000 annotated images. Achieved a validation mAP@0.5 of 0.89, which is promising for real-time inference.
- **Data Augmentation Pipeline:** Implemented advanced data augmentation techniques in the training script, including random rotation, brightness adjustment, and Gaussian noise injection. This significantly improved the model's performance in varied lighting conditions found in the production environment.
- **Inference Optimization:** Converted the trained model to ONNX format for deployment on the target edge device (NVIDIA Jetson Nano). Optimized the inference pipeline using TensorRT to achieve real-time processing speeds (~30 FPS) required for the conveyor system.

**Issues & Fixes:**
- **Issue:** The model showed high variance in performance during initial testing. It performed well in ideal lighting but failed to detect defects consistently when lighting conditions varied or objects were partially obscured. Overfitting was suspected on the initial dataset.
- **Fix:** Added significantly more diverse images to the training set, including examples of different lighting, object angles, and background clutter. The data augmentation pipeline was enhanced to synthesize these variations during training, dramatically improving generalization and reducing overfitting.

**Next Steps:**
- **Deployment & Integration:** Deploy the optimized model to the Jetson Nano and integrate it with the existing PLC communication system to trigger the rejection mechanism (actuator).
- **Edge UI:** Develop a simple dashboard on the Jetson Nano to display real-time inference results and allow operators to monitor system performance.
- **Model Iteration:** Gather new production data (false positives/negatives) to continue iterating and improving model accuracy in future training cycles.
